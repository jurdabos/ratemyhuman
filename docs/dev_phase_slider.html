1   Problem & Goal

The marketing department needs a tool that detects the emotional state of viewers watching advertisements — from a single face image.

Rather than classifying granular emotions, we adopt a threefold valence model — Negative Neutral Positive — providing a cleaner, more actionable signal.

System output per image

  • Valence label (negative / neutral / positive)
  • Confidence score for the prediction
  • Full 7-class emotion distribution for transparency

Key design decisions

  • Pretrained model — leverage state-of-the-art weights
  • Decoupled pipeline — detection, inference and mapping as independent stages
  • Built-in validation — metrics as a core component

4   Classification Pipeline

Three-stage pipeline as proposed in the concept note:

1 Face Detection
MTCNN
→
2 Emotion Inference
ViT (85.8M params)
→
3 Valence Mapping
7-class → 3-class

Stage 1 — Face Detection

  • MTCNN from facenet-pytorch
  • Crops & aligns largest face to 224×224
  • Returns None when no face found

Stage 2 — Emotion Inference

  • ViT: trpakov/vit-face-expression
  • 7-class softmax, GPU-accelerated (CUDA 12.8)

Stage 3 — Valence Mapping

  • P(Neg) = P(angry) + P(disgust) + P(fear) + P(sad)
  • P(Neu) = P(neutral)
  • P(Pos) = P(happy) + P(surprise)

Design rationale

  • Each stage independently testable and replaceable
  • Mapping is transparent and auditable
  • Surprise → Positive: fixed to advertising context

5a   Dataset — FER2013

Overview

  • Source: Kaggle — FER-2013
  • Size: ~35,887 grayscale 48×48 images
  • Labels: 7 emotion classes
  • Splits: train (~28k) / val (~3.5k) / test (~3.5k)
  • Known issue: ~65% inter-annotator agreement
  • Versioned with DVC, stored on AWS S3

Why all three splits?

The pretrained ViT was fine-tuned on the train split by its original authors — we evaluate on test only as an independent benchmark.

  • Bias context — training distribution reveals inherited biases (e.g. "happy" dominates, "disgust" ~1.5%)
  • Evaluation integrity — proves test images were unseen during training
  • Structural imbalance — class imbalance is intrinsic to FER2013, not a sampling artefact

5b   Valence Distribution

The 7 emotion classes are aggregated into 3 valence classes per the concept note mapping:

Negative

  • angry
  • disgust
  • fear
  • sad

Neutral

  • neutral

Positive

  • happy
  • surprise
Negative dominates all splits (~47%), then Positive (~36%) and Neutral (~17%)
— hence we report both macro and weighted F1.

6   Pretrained Model Integration

Model card

PropertyValue
Modeltrpakov/vit-face-expression
ArchitectureVision Transformer (ViT)
Parameters85.8 M
Input224 × 224 RGB
Output7-class softmax
Fine-tuned onFER2013

Integration approach

  • Loaded via ViTForImageClassification.from_pretrained()
  • Preprocessing via ViTImageProcessor.from_pretrained()
  • Lazy singleton in Gradio
    — model loaded on first request
  • All inference in torch.no_grad() context

Face detection

  • MTCNN — selects largest face, 20 px margin, 224×224 resize
  • PyTorch-native — same GPU device
    as the ViT model
# Minimal usage (model.py)
detector = ValenceDetector(device="cuda")
result   = detector.classify("face.png")
print(result)  # → Valence: Positive (93.2%) | Top emotion: happy (90.1%)

9   Iterative Optimisation & Summary

  • GPU environment: Resolved sm_120 compatibility with PyTorch nightly + CUDA 12.8
  • MTCNN tuning: margin=20, select_largest=True, post_process=False for low-res FER2013
  • DVC workflow: Automated push handles detection, staging, hook retries, amends
  • Gradio: Lazy-loaded singleton detector;
    auto-predict on image change
  • Validation: Added MCC, baseline comparisons, misclassified-sample visualisation
